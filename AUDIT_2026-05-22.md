## SECTION 3 — INSTITUTIONAL OPS GAPS (the big delta vs hedge-fund grade)

These are the controls every prop shop, market maker, or hedge fund running production AWS workloads has, and JustHodl.AI does not.

### 3A. No Dead-Letter Queues (0/399 = 0%)
**Risk:** When a Lambda invocation fails, the error vanishes into CloudWatch logs. There's no record of the *input* that caused failure, no replay capability, no operator visibility.

**Fix:** Add an SQS DLQ to every Lambda. One shared DLQ per family is fine. Hedge-fund standard is one DLQ per service tier (critical / important / best-effort).

```yaml
# Template — add to every Lambda config
DeadLetterConfig:
  TargetArn: arn:aws:sqs:us-east-1:857687956942:justhodl-dlq-default
```

### 3B. No distributed tracing (4/399 = 1% on X-Ray)
**Risk:** When `daily-report-v3` takes 262 seconds (p99 from live audit), you can't see where time is spent. Was it FRED? FMP? Claude? S3? You're flying blind on latency.

**Fix:** Enable X-Ray on every Lambda (`TracingConfig: Active`). Cost is negligible. Provides waterfall views of every external call.

### 3C. No CloudWatch alarms (5/399 = 1%)
**Risk:** When `portfolio-catalysts` runs at 76.7% error rate, you only know because I just told you. There's no alert.

**Fix:** Minimum baseline (institutional standard):
- Error rate >5% over 5 min → SNS to email
- Throttle count >0 → SNS
- Duration > 80% of timeout → SNS
- Invocation count = 0 over period > 2× expected cadence → SNS (catches dead schedulers)

Generate alarms programmatically via CloudFormation or a single Lambda that iterates all 399 functions.

### 3D. No structured logging (332/399 = 83% missing)
**Current state:** Most Lambdas use `print()` statements.

**Fix:** Adopt AWS Lambda Powertools or aws-lambda-logging. JSON output with consistent fields:
```python
import aws_lambda_powertools
logger = aws_lambda_powertools.Logger(service="justhodl-X")
logger.info("processed", extra={"records": 503, "duration_ms": 412, "trace_id": ctx.aws_request_id})
```
This makes CloudWatch Logs Insights queryable like a database, which is how institutions run production debugging.

### 3E. No retry / backoff on external API calls (297/399 = 74% missing)
**Risk:** FRED 429s, FMP transient errors, Anthropic rate limits — all silently fail the run. The Khalid memory specifically notes this caused the FRED 429 silent-fail bug, fixed only in Secretary v2.2.

**Fix:** Standardize on `tenacity` or `boto3 standard retry mode`:
```python
from tenacity import retry, wait_exponential, stop_after_attempt
@retry(wait=wait_exponential(min=1, max=30), stop=stop_after_attempt(5))
def call_fred(...): ...
```

### 3F. Bare `except:` clauses (43 Lambdas)
**Risk:** Swallows `KeyboardInterrupt`, `SystemExit`, and the actual root-cause exception. Hedge-fund standard: never bare except; always `except Exception as e: logger.exception(e)`.

**Fix:** Sweep + replace in single PR.

### 3G. No reserved concurrency on hot paths (37/399 = 9%)
**Risk:** A bug in one Lambda can consume all 1,000 default concurrency, starving the rest.

**Fix:** Set reserved concurrency = 1 on all schedule-driven Lambdas (most should be) and a sensible cap on hot ones (`stock-screener`, `ai-chat`).

### 3H. No CI/CD staging environment
**Current:** Push to main → deploy to prod. No canary.

**Fix:** Two-stage deploy:
1. Push deploys to `justhodl-{name}-staging` first
2. Run verify-staging.yml → if green, deploy to prod  
3. Optional 5–10% traffic canary via Lambda alias weight (already supported by AWS — just not used)

### 3I. No SLOs / error budgets
**Fix:** For each top-15 Lambda by criticality, define:
- Availability SLO (e.g., 99.5% over 30d)
- Latency SLO (e.g., p99 < 30s)
- Freshness SLO (e.g., output <2× schedule cadence)

Track in a single dashboard. When error budget burns >50%, freeze deploys to that family until investigated.

### 3J. No secrets manager
**Current:** API keys in Lambda env vars (visible to anyone with `lambda:GetFunctionConfiguration`).

**Fix:** Migrate to AWS Secrets Manager. Lambda fetches at cold start. Rotation becomes a one-click operation. Hedge-fund standard.

## SECTION 7 — INSTITUTIONAL FEATURE GAPS (what hedge funds have that you don't)

These are not bugs — they're capabilities a prop shop would consider table-stakes.

### 7A. Point-in-Time (PIT) data integrity
**Status:** `backtest-engine` exists, but every Lambda reads "current" S3 state. No vintage data layer.  
**Add:** A `data/_pit/{yyyy-mm-dd}/` snapshot layer that backtest engines read from. Ensures no look-ahead bias in research.

### 7B. Signal lineage / model registry
**Status:** Every signal version is baked into Lambda code. No lineage from input data → engine version → output signal → backtest performance.  
**Add:** A `signal-registry` Lambda that emits structured metadata on every signal write:  
```json
{"signal_id": "earnings-cascade", "version": "1.0.3", "engine_sha": "abc123",
 "input_hashes": {...}, "params": {...}, "ts": "..."}
```
Stored as Parquet in S3 partitioned by date. Lineage-queryable via Athena.

### 7C. Time-series database for signal history
**Status:** Signal history is appended JSON files. Querying historical signal state requires reading entire blobs.  
**Add:** Migrate signal histories to **TimeStream** (AWS-native) or **DynamoDB with time-series schema** (`PK=signal_id, SK=timestamp`). Quant research becomes 100× faster.

### 7D. Backtest-as-a-service
**Status:** Manual or per-engine backtest, not centralized.  
**Add:** `justhodl-backtest-service` — a Lambda that takes (signal_id, start, end, universe) and returns Sharpe, Sortino, max-DD, hit-rate, capacity-adjusted return. Powers a `backtest.html` cockpit where you click any signal and see its performance.

### 7E. Signal orthogonality matrix (live)
**Status:** `signal-orthogonality` Lambda exists. Verify it builds a full pairwise correlation matrix across all 33+ Retail Edges engines + 10 confluences + 6 fusion meta + Pro Pack v3. Hedge funds run this nightly.  
**Add (if not done):** Output a square correlation matrix → page renders heatmap → flags any |ρ|>0.7 as redundant.

### 7F. Live PnL attribution by signal
**Status:** `pnl-attribution`, `pnl-tracker` exist. Verify they attribute realized PnL back to specific signals (not just gross PnL).  
**Add:** Per-signal Sharpe contribution. Allocator weight = f(rolling 60d signal Sharpe / signal orthogonality penalty).

### 7G. Capacity / liquidity-adjusted alpha
**Status:** No tradable-size analysis seen.  
**Add:** For each signal output, also output: max position size before signal alpha degrades by 50% (function of ADV / spread / signal type). This is what separates academic backtest from tradable strategy.

### 7H. Cost-attribution dashboard
**Status:** No Lambda-cost tagging visible.  
**Add:** AWS Cost Explorer with `User:Family` tag on every Lambda. Page shows per-family $/day. When a family burns 10× its peer median, investigate. Per memory, cost-optimization ops 748-749 already cut ~36k invocations/month — this would automate that workflow.

### 7I. Alpha decay monitoring
**Status:** `outcome-checker` exists. Verify it detects when a signal's hit-rate is decaying month-over-month (= alpha crowding).  
**Add (if not done):** Per-signal half-life estimate. When hit-rate decays below random-walk over 90d → page flags "DECAYED" → allocator stops weighting.

### 7J. Adversarial robustness / regime-shift detection
**Status:** Implicit in `regime-conditional-router`.  
**Add:** Statistical regime-shift detector that triggers a "PAUSE_NEW_SIGNALS" flag when realized vol of vol > 99th percentile. All confluence engines respect the flag.

### 7K. Data-quality SLI (statistical, not just freshness)
**Status:** Freshness-only.  
**Add:** Per data feed, also track: null %, schema drift, distribution-shift (KL divergence vs 30d baseline). Alert on >3-sigma drift. Great Expectations-style.

### 7L. Quant research workbench
**Status:** All research happens via ops/pending scripts.  
**Add:** SageMaker Studio (or even an EC2 Jupyter instance) wired to the same S3 + Athena. Lets you prototype signals in notebook BEFORE promoting to a scheduled Lambda. Standard practice at Two Sigma, Renaissance, AQR.

### 7M. Tick / intraday data layer
**Status:** EOD + 15-min crypto.  
**Add (if alpha targets are short-horizon):** Polygon WebSocket → Kinesis → DynamoDB. For longer-horizon (your main play), skip — but flag the gap explicitly.

### 7N. Alternative data ingestion plane
**Status:** Has insider trading, 13F, Google Trends, GDELT. Could expand:  
- Satellite (parking-lot foot traffic — Orbital Insight)  
- Card spend (SpendIQ, Earnest)  
- App downloads (Apptopia)  
- Freight rates (FreightWaves SONAR)  
- Job postings momentum (Revelio, LinkUp)  

Each is a $10–50k/year data sub for one signal. Consider one carefully chosen ingest.
