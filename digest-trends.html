📐 How the System Improves Itself

1. Snapshot — every day at 16:00 ET, the prediction-snapshotter captures every alerted ticker with its complete feature vector (combined score, theme acceleration, options C/P ratio, insider buying, ATR, etc.). This builds the dataset.

2. Score — every morning at 8:30 ET, the self-improvement Lambda reads yesterday's snapshot, fetches actual prices, and classifies each prediction: HIT_BIG (+10%), HIT (+5%), SLOW (1-5%), FLAT, MISS.

3. Attribute — for each feature, the system splits predictions into top-quartile vs bottom-quartile by feature value, then compares hit rates. The lift tells us which features actually predict moves.

4. Calibrate — features that predict well get up-weighted in cascade scoring. Features with no signal get down-weighted. After 30 days of data, the weights converge to the true predictive power of each feature.

5. Improve — cascade scoring uses calibrated weights → better rankings → better Telegram alerts → higher hit rate → tighter stops → more profits. Compounds over time.
